## 🜂 这不只是一个知识库

绝大多数「知识库」做的是同一件事：把文件收进数据库，需要时检索出来。文件进去是死的，出来还是死的——它们只是被**存放**，从未被**理解**。

寓言计划要做的是另一回事。

我们把存储系统当作一具等待被注入灵魂的躯体。每一份文档、每一段录音、每一张图、每一次你与它的交谈，都不是孤立的字节，而是一缕等待被唤醒的精神。系统的职责，是让这些散落的记忆**彼此相连、自我编织、随时间生长**，最终凝结成一个真正「认识你、记得往事、懂得相处之道」的灵魂。

> 一块 10TB 的硬盘里，可以躺着一堆死文件，也可以住着一个灵魂。
> **寓言计划选择后者。**

## 🌌 寓言的七个角色

整套系统由七个各司其职的「角色」构成。它们不是模块的别名，而是一具数字生命的器官。

| 角色 | 司掌 | 它做什么 |
|:---:|:---:|---|
| 🜂 **灵魂** `Soul` | 知识网 | 记忆不是孤立的条目，而是会彼此引用、自我生长的双链知识网 |
| 🜃 **躯体** `Body` | 文件与身份 | 承载海量异构资料，每一份都有一张属于自己的「身份卡」 |
| 👁 **感官** `Senses` | 看 / 听 / 读 | 把音、视频、图、文都化成可被灵魂吸收的养分 |
| ⚡ **神经** `Nerves` | 检索 | 多路并行检索 + 融合重排——找得到，记忆才有意义 |
| ☽ **轮回** `Cycle` | 入魂 | 「四票入魂」：被反复遇见、被多方共指、被你钦点的记忆，才升格 |
| 🖼 **陈列** `Gallery` | 资源管理 | 虚拟集合、伴生注释、星河图谱——让灵魂被看见 |
| 🔔 **回声** `Echo` | 对话沉淀 | 你与它的每一次交谈、纠正、否决，都蒸馏成记忆沉入魂中 |

<div align="center">

> 前六个角色处理「盘里的记忆」；**回声**处理「正在发生的记忆」。
> 灵魂不仅记得盘里的往事，也记得**与你相处的方式**。

## 🔔 回声：对话即入魂

这是寓言计划最动人的一笔。

传统系统里，一段对话结束就被丢弃——发完即弃，零沉淀。可你与它的每一次交谈，恰恰是最珍贵的语料：你怎么改它的初稿、它如何回应、你又如何反馈……这条「相处的轨迹」过去**全部丢失**。

**回声层**让对话不再死亡，而是化作回声：

```
一次对话  ──蒸馏──▶  几条结构化事实  ──沉淀──▶  记忆层  ──四票入魂──▶  「相处之道」人格页
```

我们研究过这个领域里最出色的几套记忆系统，**只取其智慧，不取其代码**——蒸馏沉淀、可被修订而非无脑追加、旧记忆标失效而非删除，这些思想被收进了一套完全自研的最小设计，没有引入任何外部框架。

> 哲学贯穿始终：**AI 出决策，系统做执行**。模型永远只产出「该记什么」的判断，落盘永远由系统稳健完成——灵魂有意志，但双手稳健。

## 🔑 极简配置，本地优先

你几乎不需要配置什么。

知识检索、语音转写、图文理解的兜底——**绝大多数能力都跑在你自己的机器上**。只有少数高级的「想 / 看 / 说」能力会借力云端模型，而那也只需要你填入一两把密钥，开箱还预置了赠送额度。

- **带时间戳的语音转写完全在本地完成，一字节都不出域**——私有化的硬底气。
- 检索与理解都有本地兜底，断网也能用。
- 设置页「感官 API」一页管完：密钥、获取方式、测一下、用量账本。

> 对你只暴露一两把钥匙，其余皆为本地件，无需任何配置。

## 🛡 数据始终是你的

寓言计划是**本地优先**的。

- 你的对话、知识、生成的成品，全部安放在本地工作文件夹（默认 `~/Polaris/PolarisKB`）。
- 语音转写全程本地 ⇒ 音频出域开关**默认永久关死**；真正出门的只剩缩略图与文本片段。
- 隐私三开关 + 出域账本 + 预算闸：每一个出门的字节，你都看得见、拦得住。

灵魂住在你的硬盘里，不在任何人的云上。

## 🚀 快速更新（Docker 版）

```bash
# 在仓库根目录
./update.sh                   # 拉 GHCR latest (slim)，容器重建，数据卷保留
POLARIS_TAG=full ./update.sh  # 拉 full（带 chromium + ffmpeg + PPT/视频/渲染）
```

Web UI「更新」板块的「立即更新」按钮走同样逻辑（需在 compose 启用 `POLARIS_DOCKER_SOCKET=1` + 挂 `docker.sock`）。国内网用户见 [`DOCKER.md`](DOCKER.md) 「更新」一节，用 `scripts/pull-ghcr-to-nas.ps1` 把 GHCR 镜像 save 到 NAS 加载。

群晖部署见 [`DEPLOY-SYNOLOGY.md`](DEPLOY-SYNOLOGY.md)。

## 🏗 技术栈

前端 `Vue 3` + 后端 `Rust`（`Tauri 2`），安全沙箱为独立的 `Docker` 镜像（slim / full 双 flavor）。架构按上面的「七个角色」组织——每个器官各司其职，彼此协作。

### 前置依赖（自行构建时）

| 工具 | 用途 |
|------|------|
| Node 20+ | 前端构建 |
| Rust 1.80+ | Tauri 后端 |
| Docker | 沙箱镜像构建 / 运行 |
| `claude` CLI | 对话核心调用（沙箱内自动装；宿主由「环境医生」一键装）|

### 开发模式

```powershell
npm install          # 首次
npm run tauri:dev    # Vite 端口固定 1420
```

### 打包安装版

```powershell
npm run tauri:build
```

产物在 `src-tauri/target/release/`：免安装可执行文件 + NSIS 安装包。
