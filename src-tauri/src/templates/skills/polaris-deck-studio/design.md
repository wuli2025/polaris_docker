# Polaris HTML-PPT 设计规范（design.md）

> 这份文档是**所有版式（含 `.funnel`）和主题都必须遵守的硬约束**。
> 写 deck.html / 加新版式 / 调主题前先读它。规则之间冲突时，**第 0 条对比度铁律永远优先**。

---

## 0. 对比度铁律（最高优先级，绝不可破）

**永远不要让深色文字压在深色背景上，也永远不要让浅色文字压在浅色背景上。**

文字必须和它脚下的背景拉开足够明度差，否则等于没字。具体：

- **深底 → 必须浅字**：深色主题 / 深色卡片 / 深色渐变块上的所有文字，用 `--text-1`(主) / `--text-2`(辅) / `--text-3`(弱)，**禁止**手写深色 hex（`#111`、`#333`、`rgba(0,0,0,…)`）。
- **浅底 → 必须深字**：浅色主题 / 白卡 / 浅色渐变块上的所有文字同理，**禁止**手写浅色 hex（`#fff`、`#eee`、`rgba(255,255,255,…)`）。
- **量化门槛**：正文/小字与背景对比度 ≥ **4.5:1**；大字（≥24px 或加粗 ≥19px）≥ **3:1**。拿不准就按 4.5:1 走。
- **小字尤其危险**：`.fs-tag`、`.fs-rate`、`.fs-foot`、`.pill`、页脚、备注这类小字，明度差要更狠——它们本来就小，再低对比就彻底糊。**这正是用户反复强调的点：小字绝不能和背景同明度。**

### 为什么用 token 就天然安全
`base.css` 的 `--text-*` 和 `--bg`/`--surface` 是**同一套主题里成对定义**的：深色主题把 `--text-1` 设成近白、`--bg` 设成近黑；浅色主题反过来。所以**只要你只用 token、不写死颜色，对比度就自动成立**。出问题的永远是“图省事手写了一个固定颜色”。

```css
/* ✅ 对：跟随主题，深浅自动反相 */
color: var(--text-1);            background: var(--surface);
/* ❌ 错：固定深字，碰到深色主题直接消失 */
color: #1a1a1a;                  background: var(--surface);
/* ❌ 错：固定白字，碰到浅色主题直接消失 */
color: #fff;                     background: var(--surface);
```

## 1. 颜色纪律

1. **只用 token，不写死十六进制**（渐变字 `--grad`、装饰描边等明确的装饰例外除外，且装饰不承载可读文字）。
2. **文字色与背景色必须取自同一主题**：别把 A 主题的字色配 B 来源的底色。
3. **`color-mix` 调出来的背景要回测对比**：像 `.funnel-step` 用 `color-mix(in srgb, var(--accent) X%, var(--surface))` 染色——accent 占比拉太高时，底色会朝 accent 偏移，可能与 `--text-1` 撞明度。**所以 funnel 的 accent 占比封顶 22%**（见 base.css），保证底色仍贴近 surface、文字仍清晰。新版式若也用 color-mix 染底，同样要压低占比并目测最深/最浅两端。
4. **强调色当文字时要够沉**：`--accent`/`--good`/`--warn`/`--bad` 直接当文字时，只用在浅底上的小面积强调（如 `.fs-rate` 的绿字配浅绿底）；深底上若要彩字，用主题里已调亮的 accent，别用原始饱和色。
5. **渐变文字（`.gradient-text`）只用在大标题关键词**，且渐变两端都要能从背景里“浮出来”；不要拿渐变去写正文或小字。

---

## 2. 字阶与层级

- 一份演示固定 4 个量级：**封面超大标题 / 页标题 / 正文 / 大数字**，档差拉开，别让两级字大小接近。
- 字重也分层：标题 800、副标题 600–700、正文 300–400、弱说明 400 配 `--text-3`。
- 用**明度（颜色 token）+ 字号 + 字重**三者一起做层级，不要只靠加大字号。

---

## 3. 间距与防重叠（结构层面的“别压字”）

对比度解决“看不清”，布局要解决“叠在一起”：

1. **同一信息块内的标签、数字、脚注走正常文档流**（flex 列 / 行），**不要用 `position:absolute` 把小标签盖到大数字上**。`.funnel` 的教训：早期把段标签/转化率绝对定位到卡片角，短卡片上就会压到大数字——已改为统一进 `.funnel-head` 顶行内排布。
2. 角标/徽标类绝对定位元素，必须给主内容留出等量 padding，且**绝不与文字重叠区**。
3. 卡片/漏斗段设了 `overflow:hidden` 时，要保证最矮的那一档也能容下全部内容（`.funnel` 最矮档 = 46% 高，已实测能放下 头行+名称+大数字+脚注）。
4. 留白优先：一页只讲一件事，字少、字大、留白多。

---

## 4. 动效

- 入场统一 `anim-fade-up` + 列表 `anim-stagger-list`（子项设 `--i` 错峰），克制，别页页花哨。
- 动效**只影响出现过程，不能改变终态对比度**：动画停在终值时，文字与背景仍要满足第 0 条。
- 深色主题可叠 `data-motion` 背景光晕；浅色严肃主题（`academic-paper`/`corporate-clean`/`minimal-white`）**不开**，避免干扰阅读、压低对比。
- `prefers-reduced-motion` 时动效落终值，终值同样要可读。

---

## 5. 主题选择

- 默认 `auto` = 一眼高级：优先深色/质感主题（`aurora`/`glassmorphism`/`tokyo-night` 等），深底配亮字。
- 仅学术/公文/财报/法务等严肃场景退浅色主题（浅底配深字）。
- **无论选哪种，第 0 条对比度铁律不变**：深主题别出现深字，浅主题别出现浅字。

---

## 6. 提交前自检清单

- [ ] 全文没有手写的固定文字色（搜一遍 `color:#`、`color:rgba(0,0,0`、`color:rgba(255,255,255`、`color:#fff`、`color:#000`）。
- [ ] 深色主题/深卡上没有深字；浅色主题/白卡上没有浅字（**逐页用最深和最浅的页各看一眼**）。
- [ ] 小字（标签、转化率、页脚、pill）与背景明度差够狠，眯眼仍能读。
- [ ] 用了 `color-mix` 染底的块，最深/最浅两端文字都清晰。
- [ ] 没有小标签用绝对定位压到大数字/标题上。
- [ ] 最矮的卡片/漏斗段也装得下全部内容、不溢出不截断。
