# 方案头脑风暴（先发散，再收敛）

任务有多条合理路线、或需求还没定形时，先把方案空间摊开，比较取舍，再选一条推荐 —— 别抓住第一个想法就闷头做。

## 流程
1. **澄清真实目标**：用户要解决的根本问题是什么？区分「他说的方案」和「他想要的结果」。
2. **发散 2-4 个方案**：每个给一句话定位 + 关键取舍（工作量 / 风险 / 可逆性 / 维护成本 / 对现有架构的侵入）。差异要真，别凑数。
3. **对照比较**：列出各方案在关键维度上的差别；点出隐藏成本和「看着好其实坑」的选项。
4. **给推荐**：明确推荐一个并说清为什么；不是罗列让用户自己挑。需用户拍板的关键岔路才抛问题。

## 约定
- 方案要落到「具体怎么做」，不是抽象口号。
- 复用现有轮子优先于新造；先查项目里有没有现成的。
- 把「明确不做 / 暂不做」也讲清楚，圈定边界。

## 不适用
路线唯一、或就是个明确的小改动 —— 直接做，别为难自己。
