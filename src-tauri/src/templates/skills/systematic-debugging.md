# 系统化调试（先定位根因，再动手修）

遇到 bug / 报错 / 行为不对，**不要一上来就猜着改**。先讲清「它该怎么运作、现在哪一步偏了」，定位到根因再修。

## 流程
1. **复现**：找到最小、稳定的复现路径。复现不了就先搞复现，别盲修。
2. **读真相，不靠猜**：读报错全文 + 栈、读相关代码、加日志 / 打印中间值。用 `file:line` 锚定。
3. **二分缩小**：从「输入正常 → 输出错误」之间逐段验证，找到第一个出错的环节。
4. **定根因**：能用一句话说清「因为 X，所以 Y」。说不清就还没找到，继续查。
5. **最小修复**：只改根因那一处，别顺手大改。修完跑复现路径确认真的好了。
6. **防回归**：补一条能拦住它的检查 / 断言 / 测试（若可测）。

## 反模式（别做）
- 没定位就改一堆地方碰运气（"shotgun debugging"）。
- 把症状压下去当修好（catch 吞掉异常、加 try 兜住、调大超时）。
- 改完不复现验证就说"修好了"。

## 约定
- 区分「症状」和「根因」：日志报在 A 不代表错在 A。
- 改之前先看清要改的东西原本为什么这么写，别误删有用逻辑。
