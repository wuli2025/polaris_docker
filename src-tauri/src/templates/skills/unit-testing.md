# 单元测试生成

为一段代码 / 一个函数写实用的单元测试。目标是**拦得住真 bug**，不是凑覆盖率数字。

## 怎么写
1. **先认清契约**：这个单元的输入、输出、副作用、错误情形是什么。读懂再写。
2. **覆盖关键面**：
   - 正常路径（典型输入）。
   - 边界（空 / 0 / 负数 / 超大 / 单元素 / 极长字符串）。
   - 错误与异常（非法入参、依赖失败、超时）。
   - 幂等 / 顺序 / 并发（若相关）。
3. **每个测试只验一件事**，命名说清「在什么条件下，期望什么」。
4. **隔离外部依赖**：网络 / 文件 / 时间 / 随机用 mock / fake / 注入，让测试确定、可重复、快。
5. **断言具体值**，别只断言"没抛异常"。

## 按栈选框架
- JS/TS：Jest / Vitest。Python：pytest。Rust：内置 `#[test]` + `#[cfg(test)] mod tests`。Go：`testing`。
- 跑法：`npm test` / `pytest -q` / `cargo test` / `go test ./...`。

## 约定
- 写完**实际跑一遍**，看到绿再说完成（见「完成前验证」）。
- 别为了覆盖率写无意义测试（测 getter、测框架本身）。
- 测试是文档：读测试能看懂这个单元怎么用。

## 注意
这是**按需**写测试，不是强制 TDD 先写测试再写实现。对难以单测的代码（大量 IO / UI / 外部进程），如实说明并改用其它验证方式，别硬凑。
